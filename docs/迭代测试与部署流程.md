# CloudLens 迭代测试与部署流程

**目标**：每次改代码后，先对**远端站点**做 Chrome 测试；若有问题则登录服务器查日志，在本地修复并推送到 GitHub，再在远端服务器重新部署。

---

## 一、正确的工作流概览

```
┌─────────────────────────────────────────────────────────────────────────┐
│  1. 本地改代码 → 2. 提交并 push 到 GitHub                                 │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  3. 用 Chrome 测试「远端」https://cloudlens.songqipeng.com                 │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
            ┌───────────────────────┴───────────────────────┐
            │ 通过 ✅                    │ 失败 ❌              │
            ↓                           ↓                    │
    本次迭代完成              4. SSH 登录服务器查错误日志      │
                                      ↓                      │
                              5. 本地修复 → 回到步骤 1         │
                                      ↓                      │
┌─────────────────────────────────────────────────────────────────────────┐
│  6. 远端服务器：git pull → 重新部署（见下文）                              │
└─────────────────────────────────────────────────────────────────────────┘
```

---

## 二、本地：测试远端站点（Chrome）

测试的是 **https://cloudlens.songqipeng.com**，不是 localhost。

```bash
cd web/frontend
npm run test:chrome:remote
```

或手动指定 baseURL：

```bash
BASE_URL=https://cloudlens.songqipeng.com npx playwright test tests/dashboard-error-display.spec.ts --project=chromium
```

- **通过**：远端页面能打开，且不出现 `[object Object]` 等异常。
- **失败**：需要登录服务器看日志，并在本地修代码、推送后，在服务器上重新部署。

---

## 三、远端服务器：登录并查错误日志

1. **SSH 登录**（将 `<实例IP>` 换成实际公网 IP，如香港 18.167.167.25，见 `terraform output` 或 `docs/AWS资源与成本说明.md`）：

   ```bash
   ssh -i ~/.ssh/cloudlens-key ec2-user@<实例IP>
   ```

2. **进入应用目录**：

   ```bash
   cd /opt/cloudlens/app
   ```

3. **查看容器与日志**：

   ```bash
   docker compose ps
   docker compose logs --tail=200 backend
   docker compose logs --tail=200 frontend
   docker compose logs --tail=100 nginx
   ```

4. **根据需要查看具体错误**（例如 502、500、前端白屏）：

   - 后端/API：`docker compose logs -f backend`
   - 前端：`docker compose logs -f frontend`
   - Nginx：`docker compose logs -f nginx` 或 `cat /opt/cloudlens/app/logs/nginx/error.log`

根据日志在**本地**修改代码，然后执行「四、本地提交并推送」和「五、远端重新部署」。

---

## 四、本地：提交并推送到 GitHub

在**你的电脑**上，在项目根目录：

```bash
cd /path/to/cloudlens
git add .
git status
git commit -m "fix(web): 修复仪表盘错误展示 / 具体问题描述"
git push origin main
```

推送后，GitHub Actions 会构建并推送 Docker 镜像（若已配置）。  
若服务器是**拉镜像**部署，则到「五、远端重新部署」；若服务器是**本机构建前端**，同样在「五」里先 `git pull` 再构建。

---

## 五、远端服务器：拉代码并重新部署

在**已 SSH 登录的服务器**上，在应用目录执行。

### 方式 A：使用智能启动脚本（推荐）

```bash
cd /opt/cloudlens/app
./scripts/start.sh
```

脚本会询问是否拉取最新代码、是否重启服务；选「是」即可完成 `git pull` + 重启。

### 方式 B：手动拉代码 + Docker 重启

```bash
cd /opt/cloudlens/app
git pull origin main
docker compose down
docker compose pull    # 若使用 Docker Hub 镜像
# 若前端在服务器上本地构建：
# DOCKER_BUILDKIT=0 docker compose build frontend
docker compose up -d
```

### 方式 C：仅重启（代码已是最新）

```bash
cd /opt/cloudlens/app
docker compose down
docker compose up -d
```

部署完成后，再次在本地执行 **`npm run test:chrome:remote`** 验证远端 https://cloudlens.songqipeng.com 。

---

## 六、小结

| 步骤 | 在哪里做 | 做什么 |
|------|----------|--------|
| 测试远端 | **本地电脑** | `cd web/frontend && npm run test:chrome:remote` |
| 查日志 | **远端服务器**（SSH） | `docker compose logs backend/frontend/nginx` |
| 改代码、提交、推送 | **本地电脑** | `git add/commit/push origin main` |
| 重新部署 | **远端服务器**（SSH） | `git pull` + `./scripts/start.sh` 或 `docker compose down && pull && up -d` |

**重要**：测试对象是**远端** https://cloudlens.songqipeng.com ，迭代后先用 Chrome 测远端，再根据结果决定是否查日志与重新部署。
